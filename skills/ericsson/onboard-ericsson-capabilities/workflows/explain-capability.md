# Explain a Capability

## Entry

Use when the user requests a Quick overview or names a capability and asks what it
does, what it needs, or how to invoke it.

## Load

Load only `../references/capabilities/{selected-capability-id}.md`.

## Procedure

1. State product maturity before discussing readiness. For partial, planned, or
   unsupported entries, state that the complete capability cannot run.
2. Explain in plain language: problem solved, realistic goal prompts, information
   the Co-Worker asks for, systems/data read, possible writes, approvals, outputs,
   artifact destinations, inspection, and common recovery paths.
3. Teach follow-up language for supported scope and filters, preview versus
   execution, output format, destination, exclusions, warnings, and safe reruns.
4. Keep the answer focused on the selected entry; do not recite the catalog.
5. Ask one question: whether the user wants readiness, a demonstration, a guided
   real run, artifact help, troubleshooting, or to stop.

## Checkpoint

Capture the selected ID, maturity, topics explained, and requested next depth. Do
not infer or persist readiness from educational material.

## Exit

Exit after the overview and one next-route question, or route directly when the
user already requested the next depth.
